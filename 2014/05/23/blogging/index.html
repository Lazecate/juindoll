自建博客与媒体猜想

关于写博客的Benchmark

最近又处于走火入魔的状态，每次看到有人转发文章，就会启动Benchmark模式：

  • 首先看文章的排版，
  • 排版好看的，就研究它使用的什么平台，微信公众自带？风铃建站？还是其他微站技术。
  • 然后就链接到对方的站点分析它的微站的拓扑结构、页面跳转流程等等。
  • 甚至会研究Button配色、文章排版的格式等细节。
  • 有了好的启发，就会回去捣鼓自己的小站，进行改良。
各种博客平台之我的评语：

  1. WordPress：这是老牌的博客服务平台了，至今仍然正常运转，并且针对移动平台的普及进行了改变。

    • 优点：功能全面，模板漂亮，能适应趋势及时面向移动平台友好（后台编辑部分）
    • 缺点： 被墙，虽然可以通过发邮件来发博文，但是模板设置等至少要翻墙一次的，况且国内读者无法访问。
  2. 新浪博客：曾经繁荣辉煌过，集结了一大批名博主，有不少有质量的文章。但如今Web端属于半死状态。
    • 优点：一些以前的文章依然有检索价值，少量忠实用户仍然在使用。文章对移动端适应做的还可以，但仅仅是还可以，loading速度太慢，界面也不够美观。
    • 缺点： Web端编辑界面相当不友好，字体丑得要死，很难排出好看的排版。
  3. 网易Lofter：这个平台 的定位有点偏轻博客，自媒体友好，易用性非常高。如果能将Lofter的内容引入到其网易阅读平台，将会是一个不错的应选平台。
    • 优点：易用性高，免去很多排版的麻烦，编辑博文的界面类似Markdown，专门针对目前的主流设备如移动设备、Mac进行了改良。支持自动将博客生成APP供人下载订阅，体现自媒体友好的倾向。支持主流博客（新浪、豆瓣等等）、图片网站（Flickr）的内容迁移，所以很容易将个人散落在各个社交媒体的内容整合起来。
    • 缺点： 没有达到简约而只是简单，为了保证效果，它尽可能多地提供可用组件，也就限制了用户的发挥，基本上除了图片和文字内容，大部分博客的变化有限。另外，不支持iOS App的自动生成。不知道是否支持将Lofter的内容引入网易阅读的平台。
  4. 微信公众平台微站：结合其主旨定位，“再小的品牌也有自己的平台”，因此微站的配置也是极简的，它适合作为一个宣传的窗口，或者门户，将用户导入到其他页面。这也确实符合它的定位。
    • 优点：构建在朋友圈上的平台，用户粘稠度以及用户信息的情况比较明朗。搭建简单，适合将外链链接过来，搞一些临时活动、互动等。
    • 缺点： 页面目前大部分为静态页面，即使使用了各种微站建站的支持，其丰富度也无法与普通手机网站比拟。
  5. github静态博客：
    • 优点：永久免费server，没有被墙，github上众多的优质开源资源。只要有想法、愿意花时间折腾，可以做出功能很全面的个人站点。
    • 缺点： 需要操作性太强，除非爱折腾。即使是目前最简单的Hexo建站，也随时有可能全部从头开始配置。目前我只作为备份个人作品的一个站点，避免其他平台突然中止服务。
个人写博客的原则

写博客应该像陈寅恪的治学态度，凡是已知的、别人写过的，我不写；我写出的东西必须是自己一份努力思考加工、亲身经历实现，才写出与人分享，否则就是违背自己的立场——对于随意转发和缺乏深思批判的写作的反对立场。

对于自媒体的一些猜想：

基于朋友圈、close社交圈的自媒体
中国有13亿人，这将是巨大的媒体消费缺口。仅靠主流媒体的输出，即使再加上微博微信移动互联网等平台，也始终避免不了信息荒的问题。现在看起来似乎信息传递非常便利，又是信息爆炸的时代，但其实对于个体来说，想要获得有价值的信息反而更难了。

很快人们不会再满足于阅读普遍转发而不知真伪的内容，很快更多的人会希望更轻松地获取想要阅读的信息。这样一来，基于朋友圈、紧密小社交圈的自媒体也许会变得流行。这些小的自媒体并不是要实现争霸舆论界的野心，它们的宗旨只是服务于一个小圈子，方便圈内人的信息分享与交流。
这种基于朋友圈的好处是：

  • 兴趣点的共通性：朋友圈中的人必定有一些共性，这种方式得到的信息比随处看来的内容有更大的可能会吸引你。

  • 构建于真实人际关系的信用约束：小圈子中的人互动更频繁，谎言更容易被拆穿，这就约束信息的发布者自律。

    自媒体的“自选超市”：
    由网易、新浪、腾讯这样的大鳄提供平台，而自媒体在其中生产千千万万的独家内容，信息消费者根据自己的爱好在其中选择自己喜欢的订阅。目前网易阅读已经这么做了，只是还可以继续深化广度和力度。
